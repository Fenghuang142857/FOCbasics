What is CPU Architecture?

It is the internal logic design that determines how a processor executes instructions, manages data flow, and interacts with memory. The most common model is the Von Neumann architecture.

Figure 1: Simplified internal structure of a processor.

Core Components

ALU

The Arithmetic Logic Unit performs all mathematical calculations (addition, subtraction) and logical operations (AND, OR, NOT).

Control Unit (CU)

The "brain" of the CPU that decodes instructions from memory and coordinates the timing and movement of data between components.

Registers

High-speed, internal storage locations used to hold small amounts of data or instructions currently being processed.

Buses

Communication pathways (Address, Data, and Control) that transfer signals between the CPU, memory, and I/O devices.

Cache Memory

Small, extremely fast memory located on the CPU chip to store frequently used data and speed up the fetch-execute cycle.

Clock

An electronic oscillator that generates pulses at a steady rate to synchronize all internal operations of the processor.

Architecture Comparison

Concept Description Importance
Fetch Cycle Retrieving an instruction from main memory (RAM) First step in running any program
Decode Cycle Translating the instruction into signals for the CPU Ensures the ALU knows what to calculate
Execute Cycle Carrying out the actual instruction or calculation Produces the final output or data move
Von Neumann Shared memory for code and data Simple design used in most PCs
Harvard Separate memory for code and data Faster; common in microcontrollers
RISC vs CISC Simple instructions vs complex instruction sets Impacts power use and processing speed
